# Quick Start User Guide

## Who This Guide Is For

This guide is for first-time users who want to get productive in RGP quickly without reading the entire manual.

## Primary Navigation

The main navigation areas are:

- Requests
- Runs
- Reviews
- Promotions
- Analytics
- Admin

For most users, **Requests** is the starting point.

## Basic Request Workflow

### 1. Create a Request

Go to **Requests → New**.

Choose a published template, complete the required fields, and save or submit the request.

### 2. Submit the Request

Once submitted, the request becomes governed by its bound template version and workflow rules.

### 3. Track the Request

Use the request detail page to monitor:

- current status
- ownership
- runs
- checks
- reviews
- promotions
- agent sessions

### 4. Collaborate with an Agent if Needed

If the request supports agent execution:

- open the request
- go to the **Agents** area
- assign the request to an available agent integration
- interact with the agent session
- accept the response when satisfied to resume workflow

### 5. Review and Promotion

If your role includes review or promotion:

- use the review queue or promotion queue
- act explicitly on the request
- complete the final approval/promotion steps

## Common Roles

### Submitter

- create and submit requests
- answer questions
- interact with assigned agents
- track lifecycle progress

### Reviewer

- work review queue items
- approve, reject, or request changes

### Operator

- monitor active work, checks, and promotions
- manage workflow progression where operator action is required

### Admin

- manage templates
- manage organization and teams
- manage integrations and provider settings

## Fastest High-Value Screens

If you are new, start here:

- `/requests`
- `/requests/new`
- `/reviews/queue`
- `/analytics`
- `/admin/templates`

## Tips

- start from list pages; they are the canonical entry point
- use the request detail page as the source of truth for a single work item
- when using agents, remember that the session is interactive and persistent
- use analytics to understand trends, not to replace the request lineage
